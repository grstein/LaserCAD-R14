# LaserCAD R14 — Release Notes v1.0

Data: 2026-05-12.

## Resumo

Primeira versão estável do LaserCAD R14: micro-CAD 2D que abre via `file://` (duplo-clique em `index.html`) e exporta SVG plain para LaserGRBL. Sem servidor, sem build, sem framework.

## Funcionalidades entregues

### Sprint 1 — Specs (33 docs)
- Specs por módulo + 2 contratos de convenção (`namespace.md`, `state-contract.md`) + ADR 0001.

### Sprint 2 — MVP funcional
- Shell HTML/CSS (paleta laser 450nm, layout R14: menubar/toolbar/viewport/command-line/statusbar).
- Kernel: `vec2`, `epsilon`, `line`, `circle`, `arc`, `project` (lazy lookup ADR 0002 §1).
- Documento: `schema`, `validators`, `commands`, `history`.
- Render: `camera`, `svg-root`, `grid` (decimação por zoom), `entity-renderers`, `overlays` (crosshair).
- UI: toolbar (10 ferramentas), command-line (parser + histórico), statusbar (coords + toggles), menubar, dialogs.
- App: state singleton (12 campos, mutação centralizada), config, shortcuts, event-bus (10 eventos canônicos), bootstrap (20 passos ADR 0002 §2).

### Sprint 3 — Drawing tools
- Line, Polyline, Rect, Circle, Arc com preview ao vivo.
- Ortho lock via Shift ou toggle F8.
- Entrada por command line: absoluta `X,Y`, relativa `@X,Y`, distância.

### Sprint 4 — Precision
- Snaps: endpoint, midpoint, center, intersection (com marcadores coloridos e prioridade configurada).
- `intersect.js` com line×line, line×circle, circle×circle.
- Undo/Redo (`Ctrl+Z` / `Ctrl+Y`) com pilha de 200 comandos.
- Select tool com click-to-select e drag-to-select (box).
- Move tool (base point → destination).
- Delete (`Del`).

### Sprint 5 — Edit + Export
- Trim: line×line, line×circle.
- Extend: line→line, line→circle.
- Export plain SVG com 3 presets: **cut** (vermelho), **mark** (azul), **engrave** (verde).
- `xmlns`, `width`/`height` em mm, `viewBox` em mm, `fill="none"`, stroke 0.1mm.
- Autosave em `localStorage` com debounce 800ms; restore no boot.
- File → New limpa o documento e o autosave.

### Sprint 6 — Hardening
- README.
- Atalhos documentados.
- Exemplos.
- Tratamento de erros: banner visível para falhas de namespace e de hosts DOM (ADR 0002 §3).

## Arquitetura

- **2 ADRs**: 0001 (arquitetura base), 0002 (riscos de integração).
- **Sem ES modules** — scripts clássicos sob `window.LaserCAD.*` via IIFE.
- **33 arquivos `.js`**, **3 CSS**, **1 HTML**.
- **Pureza do core** preservada (exceto `project.js` com lazy lookup explícito).

## Limitações conhecidas

- Fontes Inter/JetBrains Mono não carregam em `file://`; fallback para `system-ui` / `Menlo`. Visual permanece coerente.
- Trim de círculo por linha não implementado (foi explicitamente Sprint 5 fora de escopo).
- Sem importação SVG (Sprint futura).
- Sem fillet, chamfer, offset (fase pós-MVP).
- Sem texto vivo (LaserGRBL não suporta de qualquer forma — converter em path antes da exportação).
- `entities` armazenadas em memória + `localStorage`; sem persistência server-side.

## Compatibilidade

- Chrome / Edge / Firefox modernos em desktop.
- Safari não testado.
- Mobile não suportado no MVP.
